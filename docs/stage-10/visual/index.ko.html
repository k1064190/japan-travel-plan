10단계

사이드바 일정 아래 별점 정렬 "주변 후보" 카드 섹션 추가, 맛집만·먼 후보 필터 토글을 지도 레이어와 공유.

Sidebar Candidates

왜

Stage 8/9에서 alt 후보·맛집을 지도 마커로 노출했지만 사이드바 일정 탭에서는 보이지 않았다. 작은 마커는 폰 화면에서 정확히 누르기 어렵고, 별점을 한눈에 정렬해서 보기도 불편하다. 사이드바에 카드 형태로 보여주면 (1) 별점 내림차순으로 인기 장소를 즉시 파악, (2) 큰 터치 타깃, (3) 필터로 원하는 후보만 좁혀볼 수 있다.

무엇을

  • 사이드바 "주변 후보" 섹션: 일정 stop 목록 아래에 추가, 별점 내림차순 정렬(별점 없는 항목은 뒤에 가나다순); 카드 클릭 → selectAlt(id); 각 카드에 이름 + 태그 3개 + 인라인 별점
  • 필터 토글 2개(체크박스 inline): 맛집만(isFoodPlace — 별점 필드 있거나 음식 키워드 태그 정규식 매칭); 먼 후보도 보기(반경을 ALT_PROXIMITY_KM 2 km에서 ALT_FAR_KM 8 km로 확장해 츠루하시 코리아타운 같은 의도적으로 먼 항목 노출)
  • 지도 마커와 사이드바 카드가 동일 필터 state 공유 — 토글 한 번에 둘 다 갱신

어떻게

state에 filterFood와 showFarCandidates 두 boolean 추가. altCandidatesForDay(day, opts) 시그니처에 opts.foodOnly/opts.far 추가; renderDay와 renderSidebar 양쪽이 동일 옵션 객체로 호출. renderSidebar가 stops <ul> 뒤에 candidateSection 블록 추가, 별점 내림차순 정렬. 토글 change 이벤트는 state 업데이트 후 renderDay와 renderSidebar 둘 다 호출해 지도·사이드바 동기 갱신.

코드 위치

  • assets/app.js:137-172 (state 필드, ALT_PROXIMITY_KM/ALT_FAR_KM, FOOD_TAG_RE, isFoodPlace)
  • assets/app.js:180-205 (altCandidatesForDay opts 파라미터)
  • assets/app.js:247-251 (renderDay에서 state 옵션 전달)
  • assets/app.js:347-471 (renderSidebar 후보 섹션 + 토글 핸들러)

회고

altCandidatesForDay에 opts 파라미터만 추가하니 두 호출자가 자연스럽게 토글을 지원하게 됐다. 단일 source of truth. isFoodPlace 휴리스틱이 단순한데도 잘 동작했다 — 별점 필드가 있는 항목은 거의 식당이라 양호한 신호. Day1 첫 카드는 야끼니쿠 M(Tabelog 3.78), Day2는 야마모토 멘조(3.92).
