Stage 3

Mobile-first UX with a bottom tab bar, full-screen detail modal, active marker pulse animation, and iOS safe-area handling — the primary use case is a smartphone in the field.

Mobile Polish

Why

The final use scenario is navigating on a smartphone in Japan. The Stage 1-2 desktop layout collapses poorly on mobile, so a dedicated mobile UX is required. With 65 places now loaded, the detail panel also needs to be organized so all information fits on a single screen without scrolling through noise.

What

  • Mobile bottom tab bar (Map / Itinerary) — <nav id="mobile-tabs"> with two buttons toggled by JS
  • Mobile detail panel promoted to a full-screen modal (z-index: 1500)
  • selectStop automatically returns to map view on mobile
  • Detail panel polish: emoji gradient hero (h-40, text-7xl), rounded close button, transit card with day-color left border, tag chips, recommended activities with day color, restaurant cards (name + price range in one row, type/tip below), review link 3-column grid (Naver Blog / Google Web / YouTube)
  • Active marker pulse animation (@keyframes pulse-marker, 1.4 s ease-in-out, infinite)
  • --active-day CSS variable auto-synced on day change, driving both mobile tab icon color and marker pulse color from a single source
  • iOS env(safe-area-inset-bottom) applied to tab bar (added during Stage 3 review)

How

@media (max-width: 767px) switches the sidebar to position: absolute full-screen and the detail panel to position: fixed full-screen. setMobileView(view) toggles sidebar/map CSS classes and updates tab button active state. selectStop checks matchMedia("(max-width: 767px)") and automatically returns to map view. The active marker is set by iterating state.markerLayer.eachLayer to strip .active from all markers, then attaching it to markers[stopIndex] and calling setZIndexOffset(1000) so the pulse stays above adjacent markers.

Code Locations

  • index.html — <nav id="mobile-tabs"> markup
  • assets/styles.css — mobile media query block, @keyframes pulse-marker, .numbered-marker.active rule, safe-area-inset-bottom on tab bar
  • assets/app.js — setMobileView, renderMobileTabs
  • assets/app.js — renderDay --active-day CSS variable update
  • assets/app.js — selectStop active marker toggle + setZIndexOffset + mobile auto-close
  • assets/app.js — renderDetail (transit card, tag chips, restaurant cards, review grid)
  • assets/app.js — hashchange mobile view sync

Retrospective

  • The desktop layout was untouched: Tailwind md:hidden classes and the @media (max-width: 767px) block add mobile UX in full isolation, with a clear breakpoint boundary.
  • The single --active-day CSS variable synchronizes mobile tab icon color and marker pulse color with one JS assignment, eliminating any JS-CSS drift.
  • The eachLayer cleanup + index-based marker attachment pattern is correct and self-cleaning: clearLayers() automatically removes all active state when the day changes.
  • Stage-level review added four improvements: Leaflet z-index offset, review_links null guard, iOS safe-area inset, and hashchange mobile view sync.
  • Pulse animation runs infinitely at 1.4 s; battery impact is negligible but present. UX clarity takes priority at this cadence.
