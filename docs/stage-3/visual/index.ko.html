3단계

모바일 우선 UX 전면 개선: 하단 탭바, 전체화면 디테일 모달, 활성 마커 펄스 애니메이션, 디테일 패널 레이아웃 정리.

Mobile Polish

왜

최종 사용 시나리오는 현장에서 스마트폰으로 보는 것이다. Stage 1~2의 데스크톱 레이아웃은 모바일에서 짜그라지기 때문에 별도의 모바일 UX가 필요했다. 또한 정보가 많아진 만큼 디테일 패널이 한 화면에 들어오도록 정리가 필요했다.

무엇을

  • 모바일 하단 탭바(지도/일정) — <nav id="mobile-tabs">에 두 버튼, JS로 토글
  • 모바일 디테일 패널 → 풀스크린 모달(z-index 1500)
  • selectStop 호출 시 모바일에서 자동으로 지도 뷰 복귀
  • 디테일 패널 폴리시: emoji 그라데이션 hero + 둥근 닫기 버튼, 이동 카드(day 색상 좌측 보더), 태그 chip, 활동 목록, 맛집 카드(이름+가격 같은 행), 후기 버튼 3열 그리드(네이버 블로그/구글/유튜브)
  • 활성 마커 펄스 애니메이션(@keyframes pulse-marker, 1.4s ease-in-out)
  • --active-day CSS 변수가 day 전환 시 자동 갱신 → 모바일 탭 아이콘 색 동기화

어떻게

@media (max-width: 767px)에서 사이드바를 absolute 풀스크린, 디테일을 fixed 풀스크린으로 전환. setMobileView(view)가 sidebar/map 클래스를 토글하고 탭 버튼 active 상태를 갱신. selectStop에서 matchMedia("(max-width: 767px)")를 체크해 자동으로 지도 복귀. 활성 마커는 state.markerLayer.eachLayer로 모든 마커의 .active를 제거 후 markers[stopIndex]에만 추가하며 CSS animation으로 pulse 처리.

코드 위치

  • index.html — <nav id="mobile-tabs"> 마크업
  • assets/styles.css — 모바일 media query 블록, @keyframes pulse-marker, .numbered-marker.active 룰
  • assets/app.js — setMobileView, renderMobileTabs, renderDay의 --active-day 설정, selectStop active 마커 토글 + matchMedia 모바일 닫기, renderDetail 전체

회고

데스크톱 레이아웃을 손대지 않고 md:hidden Tailwind 클래스와 @media (max-width: 767px) CSS만으로 모바일 전용 UX를 추가했다. --active-day CSS 변수 하나로 모바일 탭 아이콘 색과 마커 펄스 색을 단일 source of truth로 동기화한 것이 핵심이었다.
