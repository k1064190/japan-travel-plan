14단계

장소 선택 시 지도 위에 링 버스트 애니메이션을 표시해 활성 마커를 즉시 시각적으로 식별할 수 있게 함.

Marker Highlight Burst

왜

Stage 13에서 "맵에서 보기" 버튼을 추가했지만 사용자가 "맵으로 이동은 되는데 그 자리에서 정확히 어디인지 찾기가 힘듦"이라고 보고했다. numbered 마커 + alt 마커가 같은 화면에 겹쳐있을 때 활성 마커를 시각적으로 즉시 찾을 수 있는 신호가 필요했다.

무엇을

  • highlightLocation(coords, color?) 헬퍼: .highlight-ring divIcon 마커를 잠깐 띄움; CSS @keyframes highlight-burst로 0.25× → 3× scale + opacity 0.95 → 0 fade, 1.4s ease-out; 1.5초 후 자동 removeLayer
  • 3개 경로에 통합: selectStop()(일정 stop 클릭), selectAlt()(alt 후보 마커/사이드바 카드 클릭), #detail-locate 클릭(flyTo 600ms 후 burst, 카메라 안착 타이밍에 맞춤)
  • ring 색상은 day color로 inline-style — 활성 day와 일관된 시각 신호
  • interactive: false + keyboard: false로 ring이 클릭을 가로채지 않음
  • zIndexOffset: 2000으로 numbered/alt 마커 위에 표시

어떻게

CSS:

@keyframes highlight-burst {
  0%   { transform: scale(0.25); opacity: 0.95; }
  100% { transform: scale(3);    opacity: 0; }
}
.highlight-ring {
  width: 60px; height: 60px; border-radius: 50%;
  border: 4px solid #3b82f6;
  pointer-events: none;
  animation: highlight-burst 1.4s ease-out forwards;
}

#detail-locate에서는 flyTo(0.8s) 이후 카메라 안착 시점에 burst가 절정에 닿도록 setTimeout(..., 600) 추가.

코드 위치

  • assets/styles.css:80-98 (@keyframes highlight-burst + .highlight-ring)
  • assets/app.js:311-330 (highlightLocation 헬퍼)
  • assets/app.js:401 (selectAlt)
  • assets/app.js:441 (selectStop)
  • assets/app.js:670-672 (#detail-locate 클릭 핸들러, 지연 burst)

회고

단일 헬퍼 + 3개 경로 연결로 일정 stop/alt 후보/detail-locate 어느 경로로 와도 동일한 시각 신호를 제공했다. divIcon + CSS animation 조합이 Leaflet circleMarker로는 불가능한 부드러운 scale·fade를 자연스럽게 처리했다. setTimeout cleanup이 map.hasLayer를 가드해 활성 day 변경으로 인한 map clear 후에도 안전하게 동작한다.
