Stage 14

A ring burst animation appears on the map whenever a location is selected, making the active marker immediately identifiable among dense marker layers.

Marker Highlight Burst

Why

Stage 13 added a “View on Map” button, but users reported that after flying to a location it was hard to find exactly which marker was active among the overlapping numbered and alt markers. A brief visual burst on the active coordinate removes that ambiguity instantly.

What

  • highlightLocation(coords, color?) helper: briefly places a .highlight-ring divIcon marker; CSS @keyframes highlight-burst animates 0.25× → 3× scale + opacity 0.95 → 0 fade over 1.4 s ease-out; auto-removed after 1.5 s
  • Wired into 3 paths: selectStop() (itinerary stop click), selectAlt() (alt candidate marker or sidebar card click), #detail-locate click (burst fires 600 ms after flyTo to match camera landing)
  • Ring color is the active day color via inline style — consistent with numbered markers and sidebar
  • interactive: false + keyboard: false so the ring never intercepts clicks
  • zIndexOffset: 2000 renders the ring above all other markers

How

CSS:

@keyframes highlight-burst {
  0%   { transform: scale(0.25); opacity: 0.95; }
  100% { transform: scale(3);    opacity: 0; }
}
.highlight-ring {
  width: 60px; height: 60px; border-radius: 50%;
  border: 4px solid #3b82f6;
  pointer-events: none;
  animation: highlight-burst 1.4s ease-out forwards;
}

For #detail-locate the 600 ms delay aligns the burst peak with the camera settling after the 0.8 s flyTo.

Code Locations

  • assets/styles.css:80-98 (@keyframes highlight-burst + .highlight-ring)
  • assets/app.js:311-330 (highlightLocation helper)
  • assets/app.js:401 (selectAlt)
  • assets/app.js:441 (selectStop)
  • assets/app.js:670-672 (#detail-locate click handler, delayed burst)

Retrospective

A single helper wired into 3 paths delivers the same visual signal regardless of how the location was selected. The divIcon + CSS animation approach handles smooth scale-and-fade that Leaflet’s circleMarker cannot. The map.hasLayer guard in the cleanup timeout keeps the code safe even if the active day changes before the ring fades out.
