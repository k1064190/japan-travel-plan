Stage 13

“View on Map” button in the detail panel closes the panel and flies to the location at zoom 18, plus a GPS self-location control in the top-left of the map.

Locate And GPS

Why

The detail panel partially covered the map, so knowing a place’s surroundings after opening the panel required an extra action. In the field, users also need to see their own position relative to candidates to judge which ones are physically closest.

What

  • Detail header “View on Map” button: closes the detail panel, calls state.map.flyTo(place.coords, 18, {duration: 0.8}), and on mobile automatically switches to map view via setMobileView("map"); works for both itinerary stops and alt candidates through the single renderDetail path
  • Leaflet custom control “GPS / My Location” in the top-left: calls navigator.geolocation.getCurrentPosition with enableHighAccuracy: true; on success places a blue dot marker with an accuracy circle and flies to zoom 16; on failure shows a Korean error message per geolocation error code (1 = permission denied, 2 = unavailable, 3 = timeout); re-clicking removes the previous marker before placing a new one
  • CSS .leaflet-control-locate: 30×30 white button, slate-100 on hover, compatible with Leaflet default control styling

How

addLocateControl() called at the end of initMap() using L.Control.extend({ position: ‘topleft’, onAdd() {...} }). L.DomEvent.disableClickPropagation separates the button click from map clicks. requestLocation() is a separate function bound to the control’s onclick. userLocationMarker and userLocationAccuracy are module-level state variables enabling cleanup on re-click. The detail header button is a <button id="detail-locate"> in renderDetail’s innerHTML with a click handler attached at the end, following the same pattern as #detail-close.

Code Locations

  • assets/app.js:224-313 (initMap → addLocateControl, requestLocation, userLocationMarker/Accuracy state)
  • assets/app.js:520-523 (detail header #detail-locate button)
  • assets/app.js:592-601 (detail-locate click handler)
  • assets/styles.css:67-78 (.leaflet-control-locate)

Retrospective

Using the Leaflet custom control pattern placed the GPS button naturally beside the zoom controls with no extra floating UI. The single renderDetail path means both itinerary stops and alt candidates get the “View on Map” button automatically. All three geolocation error codes (permission denied, unavailable, timeout) produce distinct Korean messages rather than a generic alert.
