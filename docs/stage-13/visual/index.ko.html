13단계

디테일 패널에 "맵에서 보기" 버튼 추가(패널 닫고 zoom 18 flyTo), 지도 좌상단에 GPS 내 위치 컨트롤 구현.

Locate And GPS

왜

디테일 패널이 지도를 일부 가리기 때문에 패널을 열고 나서 주변 위치를 확인하려면 추가 동작이 필요했다. 또한 현장에서 자기 위치 기준으로 어떤 후보가 가까운지 직관적으로 보려면 GPS가 필요했다.

무엇을

  • 디테일 헤더 "맵에서 보기" 버튼: 디테일 패널 닫기 → state.map.flyTo(place.coords, 18, {duration: 0.8}) → 모바일이면 setMobileView("map")으로 지도 탭 자동 전환; 단일 renderDetail 경로라 일정 stop과 alt 후보 모두 지원
  • Leaflet 커스텀 컨트롤 "내 위치 (GPS)" — 지도 좌상단: navigator.geolocation.getCurrentPosition + enableHighAccuracy: true; 성공 시 파란 점 마커 + 정확도 circle + zoom 16 flyTo; 실패 시 한국어 에러(에러코드 1 = 권한 거부, 2 = 위치 불가, 3 = timeout); 재클릭 시 기존 마커 제거 후 새로 생성
  • CSS .leaflet-control-locate: 30×30 흰색 버튼, hover 시 slate-100, Leaflet 기본 컨트롤 스타일 호환

어떻게

initMap() 끝에 addLocateControl() 호출 — L.Control.extend({ position: 'topleft', onAdd() {...} }) 패턴으로 버튼을 컨트롤로 등록. L.DomEvent.disableClickPropagation으로 맵 클릭과 분리. requestLocation()은 별도 함수로 컨트롤 onclick에 바인딩. userLocationMarker/userLocationAccuracy는 모듈 상태 변수로 두어 재클릭 시 cleanup 지원. 디테일 헤더 버튼은 renderDetail의 innerHTML에 <button id="detail-locate">를 추가하고 끝에 클릭 핸들러를 attach(#detail-close와 동일한 패턴).

코드 위치

  • assets/app.js:224-313 (initMap → addLocateControl, requestLocation, userLocationMarker/Accuracy 상태)
  • assets/app.js:520-523 (디테일 헤더 #detail-locate 버튼)
  • assets/app.js:592-601 (detail-locate 클릭 핸들러)
  • assets/styles.css:67-78 (.leaflet-control-locate)

회고

Leaflet 커스텀 컨트롤 패턴으로 zoom 컨트롤 옆에 자연스럽게 GPS 버튼을 배치했다. 별도의 floating UI가 필요 없었다. 단일 renderDetail 경로 덕분에 일정 stop과 alt 후보 양쪽에 "맵에서 보기" 버튼이 자동으로 붙었다. 권한 거부·위치 불가·timeout 세 가지 에러코드가 모두 별개의 한국어 메시지로 분기된다.
