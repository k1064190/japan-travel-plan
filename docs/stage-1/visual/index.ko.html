1단계

5개 샘플 장소로 데이터 로딩, Leaflet 지도, 사이드바, 디테일 패널, 해시 라우팅의 종단간 연결을 검증한 프로젝트 뼈대.

Skeleton

왜

전체 설계를 한 번에 만들기 전에, 데이터 → 지도 → 사이드바 → 디테일 → 라우팅의 연결이 5개 샘플 명소로 처음부터 끝까지 동작하는지 검증이 필요했다. 60개 데이터를 채운 뒤 연결 오류가 생기면 되돌리기 비용이 크기 때문이다.

무엇을

  • index.html 골격 + Leaflet 1.9.4 / Tailwind play CDN / Pretendard CDN 임포트
  • assets/styles.css — day 색상 CSS 변수(--day-1 ~ --day-5), numbered-marker 스타일, 모바일 media query
  • assets/app.js — 데이터 로더(에러 처리 포함), Leaflet 마커, 사이드바, 디테일 패널, 해시 라우터, XSS-safe esc() / safeHref() 헬퍼
  • data/itinerary.json — Day1 5개 stop
  • data/places.json — 5개 샘플(간사이공항, 난바숙소, 구로몬, 덴덴타운, 도톤보리)

어떻게

Vanilla ES6, 단일 모듈(assets/app.js). 상태는 in-memory state 객체. 지도는 fitBounds로 day 전체가 보이도록 시작, 클릭 시 flyTo. URL hash는 #dayN/placeId 포맷으로 새로고침 시에도 상태 복원. boot()는 데이터 fetch 실패 시 한국어 에러 메시지를 표시.

코드 위치

  • index.html (골격 + CDN imports)
  • assets/styles.css (day 색상 CSS vars + numbered-marker + 모바일 media query)
  • assets/app.js — DAY_COLORS / dayColor / esc / safeHref
  • assets/app.js — state, loadData, initMap
  • assets/app.js — renderDay, selectStop
  • assets/app.js — renderSidebar, renderDetail, renderProgressBar
  • assets/app.js — parseHash, boot, hashchange listener
  • data/places.json, data/itinerary.json (5개 장소, Day1)

회고

TDD 없이도 task별 spec+코드 리뷰가 중요 이슈 6개를 사전 차단했다(loadData 에러 처리, XSS escaping, null 가드, DAY_COLORS 폴백, hashchange race, aria-label). Tailwind play CDN + Pretendard로 빌드 단계 없이 미려한 UI를 확보하고, 단일 HTML 파일로 로컬 실행이 가능했다.
