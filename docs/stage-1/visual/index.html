Stage 1

Project skeleton, routing, and end-to-end wiring validated with 5 sample stops before expanding to the full 60-place dataset.

Skeleton

Why

Verify that the full data → map → sidebar → detail → routing pipeline works end-to-end with 5 sample places before populating 60 entries. Rewiring a broken pipeline after filling all data would be expensive to reverse.

What

  • index.html skeleton + Leaflet 1.9.4 / Tailwind play CDN / Pretendard CDN imports
  • assets/styles.css — day color CSS variables (--day-1 through --day-5), numbered-marker styles, mobile media query
  • assets/app.js — data loader with error handling, Leaflet markers, sidebar, detail panel, hash router, XSS-safe esc() / safeHref() helpers
  • data/itinerary.json — Day 1, 5 stops
  • data/places.json — 5 sample places (Kansai Airport, Namba accommodation, Kuromon, Den Den Town, Dotonbori)

How

Vanilla ES6, single module (assets/app.js). State is held in an in-memory state object. The map starts with fitBounds to show the full day, then uses flyTo on click. URL hash format is #dayN/placeId, restored on reload. boot() displays a Korean-language error message on data fetch failure.

Code Locations

  • index.html — skeleton + CDN imports
  • assets/styles.css — day color CSS vars + numbered-marker + mobile media query
  • assets/app.js — DAY_COLORS / dayColor / esc / safeHref helpers
  • assets/app.js — state, loadData, initMap
  • assets/app.js — renderDay, selectStop (map + markers)
  • assets/app.js — renderSidebar, renderDetail, renderProgressBar (UI)
  • assets/app.js — parseHash, boot, hashchange listener (routing)
  • data/places.json, data/itinerary.json (5 places, Day 1)

Retrospective

  • Per-task spec and code review caught 6 critical issues before implementation: loadData error handling, XSS escaping, null guards, DAY_COLORS fallback, hashchange race condition, and aria-label.
  • Tailwind play CDN and Pretendard give a polished UI with no build step; the app runs as a single local HTML file.
  • Before Stage 2: add place guards to renderSidebar / renderDetail (insurance against data typos in 60-place expansion), and fix sidebar sync on same-day hashchange.
  • Deferred to Stage 3: mobile-tabs intentionally left empty (Stage 3.1); .numbered-marker.active CSS is dead code preserved for Stage 3.3 activation.
  • Stage 2 readiness: ready with caveats — enter after applying critical guard fixes.
