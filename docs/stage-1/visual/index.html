Stage 1

Project skeleton with end-to-end wiring: data loading, Leaflet map, sidebar, detail panel, and hash routing validated against 5 sample places.

Skeleton

Why

Before building the full design, the data → map → sidebar → detail → routing wiring needed to be verified end-to-end with 5 sample locations. Fixing a wiring mismatch after adding 60 entries would be expensive to reverse.

What

  • index.html skeleton + Leaflet 1.9.4 / Tailwind play CDN / Pretendard CDN imports
  • assets/styles.css — day color CSS variables (--day-1 through --day-5), numbered-marker styles, mobile media query
  • assets/app.js — data loader (with error handling), Leaflet markers, sidebar, detail panel, hash router, XSS-safe esc() / safeHref() helpers
  • data/itinerary.json — Day 1 with 5 stops
  • data/places.json — 5 sample entries (Kansai Airport, Namba hotel, Kuromon, Denden Town, Dotonbori)

How

Vanilla ES6, single module (assets/app.js). State is held in an in-memory state object. The map starts with fitBounds to show the full day, then flyTo on click. URL hash uses the format #dayN/placeId and restores state on refresh. boot() displays a Korean error message if data fetch fails.

Code Locations

  • index.html (skeleton + CDN imports)
  • assets/styles.css (day color CSS vars + numbered-marker + mobile media query)
  • assets/app.js — DAY_COLORS / dayColor / esc / safeHref
  • assets/app.js — state, loadData, initMap
  • assets/app.js — renderDay, selectStop
  • assets/app.js — renderSidebar, renderDetail, renderProgressBar
  • assets/app.js — parseHash, boot, hashchange listener
  • data/places.json, data/itinerary.json (5 places, Day 1)

Retrospective

Per-task spec and code review caught 6 critical issues before any data was filled (loadData error handling, XSS escaping, null guards, DAY_COLORS fallback, hashchange race, aria-label). Tailwind play CDN + Pretendard delivered a polished UI with no build step, runnable as a single local HTML file.
