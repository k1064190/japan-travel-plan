Stage 4

QA pass covering all 21 stops, hash routing edge cases, iOS safe-area inset, cross-browser validation, and a full README.

QA

Why

The final guardrail before going live: (1) all 21 stops must be clickable, (2) URL sharing and invalid hashes must degrade gracefully, (3) iOS safe-area must not obscure the tab bar, (4) anyone should be able to run the app from the README.

What

  • scripts/smoke-test.md — 21-stop regression checklist (place_id table per day + routing / mobile / cross-browser items)
  • assets/app.js — hash routing normalization + detail panel auto-close logic (two issues found during smoke test) + buildHash() helper (refactored from review feedback)
  • assets/styles.css — #mobile-tabs gains padding-bottom: env(safe-area-inset-bottom, 0)
  • README.md — run / data-edit / validate / directory structure guide
  • Final review trio passed (code-reviewer-pro + gemini-subagent; Codex fell back due to MCP tool schema error)

How

A headless Chromium script cycled through all 21 stops via hash changes, confirming each detail header loaded correctly. Two routing edge cases were found and fixed: an invalid hash (#day9/foo) previously left stale UI; fixed with history.replaceState normalization and detail-panel auto-hide. Same-day stop switches and mobile viewport (390×844) were also validated. After all fixes: node scripts/validate-data.mjs → 65 places, 5 days; node --test tests/validate-data.test.mjs → 11 pass.

Code Locations

  • scripts/smoke-test.md
  • README.md
  • assets/app.js:325-336 (buildHash helper)
  • assets/app.js:344-355 (boot() URL normalization + optional chaining)
  • assets/app.js:365-400 (hashchange handler)
  • assets/styles.css:81-85 (#mobile-tabs safe-area-inset)

Retrospective

The headless smoke script caught two routing edge cases that manual review had missed. The Codex PR review (3 rounds, 16 findings) produced purely mechanical defensive fixes — all applied without ambiguity. The parseHash silent-fallback pattern is noted for improvement in future SPAs: a valid: false signal to the caller would be cleaner.
