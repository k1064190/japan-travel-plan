6단계

모든 이동 카드에 Google Maps 실시간 경로 딥링크 추가, 도보 7구간 haversine 소요시간 검증 스크립트 구현.

Realtime Transit

왜

Stage 1~5의 이동 시간은 사전 조사 기반 추정치였다. 현장에서 라피트·JR·미도스지선 같은 일본 대중교통 시간은 환승·정차·실시간 운행에 따라 크게 달라진다. 매번 좌표를 Google Maps에 직접 입력하는 대신, 각 stop의 이동 카드에서 클릭 한 번으로 출발지·도착지·이동수단이 미리 채워진 실시간 경로 페이지를 열 수 있어야 했다.

무엇을

  • UI 딥링크: 각 이동 카드 안에 "Google Maps에서 실제 경로·시간 보기 →" 버튼 추가, 클릭 시 새 탭에 미리 채워진 Google Maps 경로 페이지 오픈
  • scripts/verify-walk-times.mjs: 순수 도보 구간을 haversine 거리 × 1.25 detour factor ÷ 5 km/h로 검증, ±3분 AND ±50% 초과 시 flag; npm run verify-walk로 실행
  • 대중교통 자동 검증 미지원(OSRM 데모는 driving fallback 반환, ORS/GraphHopper는 API key 필요, Google Directions API는 유료); UI 버튼이 1차 실시간 검증 수단

어떻게

런타임에 기존 좌표 데이터로 URL을 즉석 생성하는 새 헬퍼 3개를 추가했다. 데이터 파일 변경 없음. transitModeToGoogleMode(mode)는 한국어 이동수단 문자열을 Google travelmode로 매핑. buildDirectionsUrl(originCoords, destCoords, mode)는 좌표가 유한수가 아니면 null을 반환해 URL을 안전하게 생성. getTransitEndpoints(dayId, stopIndex)는 같은 날 직전 stop 또는 이전 날 마지막 stop을 fallback으로 사용; Codex 리뷰에서 itinerary stop에 옵셔널 origin_place_id 필드를 추가해 정확한 override를 지원하게 했다.

코드 위치

  • assets/app.js:65-103 (transitModeToGoogleMode, buildDirectionsUrl, getTransitEndpoints)
  • assets/app.js:275-296 (이동 카드 딥링크 버튼)
  • scripts/verify-walk-times.mjs (haversine + 도보속도 휴리스틱)
  • package.json (npm run verify-walk)

회고

데이터 변경 0건으로 회귀 위험을 최소화했다. Google Maps 딥링크는 API key 없이 동작하며 iOS/Android에서 기본 Maps 앱으로 자동 연결된다. haversine 휴리스틱이 도보 2개 구간을 정확히 플래그해 수정(sannenzaka 5→7분, denden_town 7→10분)하는 데 충분했다.
