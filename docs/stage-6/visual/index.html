Stage 6

One-tap Google Maps deep-link on every transit card for real-time directions, plus a haversine walking-time sanity checker for the 7 walking segments.

Realtime Transit

Why

Transit times in Stages 1–5 were pre-researched estimates. In the field, times for Rapit, JR, and Midosuji Line vary with transfers, stops, and real-time conditions. Rather than manually copy-pasting coordinates into Google Maps every time, one click from each stop’s transit card should open a live directions page pre-filled with origin, destination, and travel mode.

What

  • UI deep-link: “View real route and time on Google Maps →” button inside each transit card, opening a pre-filled Google Maps directions page in a new tab
  • scripts/verify-walk-times.mjs: sanity-checks pure walking segments using haversine distance × 1.25 detour factor ÷ 5 km/h; flags segments exceeding ±3 min AND ±50%; npm run verify-walk
  • Transit verification policy: automated verification unsupported (OSRM demo returns driving fallback; ORS/GraphHopper require API keys; Google Directions API is paid); the UI button is the primary real-time verification tool

How

Three new helpers generate the URL at runtime from existing coordinate data — zero data file changes. transitModeToGoogleMode(mode) maps Korean mode strings to Google travelmode values. buildDirectionsUrl(originCoords, destCoords, mode) builds the https://www.google.com/maps/dir/ URL safely (returns null if coordinates are not finite numbers). getTransitEndpoints(dayId, stopIndex) resolves origin from the same-day previous stop or, as a fallback, the last stop of the previous day; the Codex review prompted adding an optional origin_place_id field to itinerary stops for precise overrides.

Code Locations

  • assets/app.js:65-103 (transitModeToGoogleMode, buildDirectionsUrl, getTransitEndpoints)
  • assets/app.js:275-296 (transit card deep-link button)
  • scripts/verify-walk-times.mjs (haversine + walking-speed heuristic)
  • package.json (npm run verify-walk)

Retrospective

Zero data changes — the URL is generated at runtime from existing coordinates, minimizing regression risk. The Google Maps deep-link works without any API key and triggers the native Maps app on iOS and Android. The haversine heuristic was sufficient to flag 2 walking segments that needed adjustment (sannenzaka 5→7 min, denden_town 7→10 min); no further changes were needed.
