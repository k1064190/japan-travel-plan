Stage 16

Usj Interior

Why

Stage 4 treated USJ as a single stop, but in practice it holds 11 zones and 50+ attractions. Showing an entire Day 4 as one pin makes it impossible to plan what to do inside USJ. Korean tourists typically need advance knowledge of popular rides (Mario Kart, Forbidden Journey, Hollywood Dream), Nintendo timed-entry tickets, and seasonal events (Cool Japan, Detective Conan collaboration) to optimize routing and queue times. The goal was to model USJ internals as individual attraction entries, navigable from the map and detail panel.

What

  • Three optional schema fields added:
    • place.parent — references the parent place id to mark an entry as a child
    • place.park_zone — free-text zone label for display
    • place.attraction_type — enum ride / show / food / shop / event / landmark
  • 52 USJ child entries added — 16 rides / 13 events / 12 food / 6 shops / 4 shows / 1 landmark.
  • Zone distribution: Super Nintendo World (8), Wizarding World (8), Minion Park (5), Jurassic Park (5), Hollywood (5), New York (5), Universal Wonderland (5), USJ common events (5), Amity Village (3), San Francisco (2), WaterWorld (1).
  • 5 closed / season-ended attractions preserved in detail with a "currently closed" note: Spider-Man 4K3D (closed 2024-01-22), Backdraft (closure announced 2023-05), Sesame Street Fun World (closed 2026-05-10 — 12 days before the trip), Demon Slayer XR Ride (ended 2025-01).
  • Child coordinates placed within the USJ footprint [34.665–34.668, 135.431–135.434] using zone center ±5–10 m offsets (exact per-attraction coordinates are not published by USJ).
  • UI — state.usjLayer: child markers auto-shown when Day 4 is active. Zone-colored 22 px pins render beneath numbered markers.
  • Detail panel: "USJ Interior (52)" section inside the USJ parent page — zone-colored card groups with attraction_type badge and rating.
  • Child detail view: "← Back to USJ" back-nav + zone color badge.
  • Validator: verifies parent points to a root place (not itself), exempts children from the restaurants required-field check, enforces attraction_type enum.

How

  1. 4-subagent parallel curation (Stage 11/12/15 pattern):
    • Group 1: Super Nintendo World (8) — USJ official Korean + Wikipedia + Namuwiki + TDR Explorer
    • Group 2: Harry Potter + Minion (13) — USJ official + Wikipedia EN + KKday Korea + TDR Explorer (Hogwarts Magical Night season check)
    • Group 3: Hollywood + New York + San Francisco + Wonderland (17) — closure/reopening cross-check using Inside Universal and WDW News Today reports
    • Group 4: Jurassic + WaterWorld + Amity + common events (14) — precise 2026-05 operational status verification (Cool Japan / Conan collab through 6/30; Halloween Horror Nights and Wonder Christmas out of season)
  2. Each subagent output to /tmp/usj-patches/group{N}.json; merged by /tmp/merge-usj.mjs (first entry wins on duplicate placeId; backfills restaurants: [] / tags: [] / review_links).
  3. Validator extended: parent reference check + attraction_type enum + restaurants exemption when parent is present. Three new tests added (passing child / unknown parent / invalid type).
  4. UI: new state.usjLayer layered after routeLayer / altLayer; drawChildMarkers(parentId, color) called inside renderDay; renderUsjChildren renders zone-grouped card grid inside the detail panel; selectAlt(id) reused as the single entry point for map pin click, card click, and back-nav.

Retrospective

What went well

  • 4-subagent parallel curation completed 52 entries in ~28 minutes. Group 3 verifier correctly identified 5 closed/ended attractions as of 2026-05-21 (e.g., Sesame Street closed 11 days before the trip).
  • A single parent field cleanly represents the children / restaurants / zone tree — absorbed into places.json without a new file.
  • zoneColor prefix matching naturally falls back to the parent color for sub-zone names such as "Super Nintendo World / Donkey Kong Country".
  • Validator added optional fields non-destructively — 169 existing places regressed zero, following the same additive pattern as Stage 15.

Carry forward

  • Exact attraction coordinates are not published by USJ; pins fall somewhere within the zone but are not precise. OSM contributor data for USJ interior polygons/nodes would enable a follow-up correction.
  • Closed attraction display — currently only a text note in the detail body; a distinct visual indicator (gray pin, "closed" badge) would be clearer.
  • Timed-entry / Express Pass requirement is tag-only; linking it to the Stage 15 booking infrastructure (required badge + advance_days) would improve consistency.
  • Day 4 itinerary remains 1 stop. If user demand grows, USJ could be split into time-block sub-stops (morning = SMW, afternoon = HP, evening = parade).

Validation

  • node scripts/validate-data.mjs → 221 places, 5 days (169 + 52)
  • node --test tests/validate-data.test.mjs → 14 pass (11 + 3 new cases)
  • node /tmp/merge-usj.mjs → 52 entries added, 0 duplicates after final run
  • Data distribution confirmed: 16 ride / 13 event / 12 food / 6 shop / 4 show / 1 landmark
  • 5 closed/season-ended entries verified with "currently closed" notation in detail view
