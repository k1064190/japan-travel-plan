8단계

각 day stop 1.5 km 반경 내 alt 후보를 지도에 작은 회색 마커로 표시하고, 한국어 큐레이션 링크가 포함된 신규 alt 27개 추가.

Alt Candidates

왜

지금까지는 일정 stop만 지도에 표시됐기 때문에 places.json의 60+ alt 카테고리 후보가 UI에서 완전히 보이지 않았다. 사용자가 일정대로 이동하면서 "주변에 뭐가 있는지" 앱을 벗어나지 않고 즉시 확인할 방법이 없었다.

무엇을

  • UI: 활성 day의 일정 stop 중 하나라도 1.5 km 이내인 alt 카테고리 명소를 작은 회색 원형 마커로 표시; 호버 시 명소명 툴팁, 클릭 시 디테일 패널("후보 — 일정에 들어있지 않음" 라벨, 시간 없음)
  • 데이터: 4개 background subagent가 day별 area로 신규 alt 27개 추가(Day1: 7개 카페/신사/야식; Day2: 8개 교토 동쪽 사찰/% Arabica/철학의 길; Day3: 8개 오사카 미술관/키타하마 카페/코리아타운; Day4: 4개 덴포잔 공원/시티워크/환승역)
  • itinerary 변경 0건(사용자 요구)

어떻게

renderDay가 stops를 그리기 전 altCandidatesForDay(day)를 호출 — 활성 day의 stop 중 하나라도 1.5 km 이내인 모든 alt 카테고리 장소를 반환(현재 day의 stop 자체는 제외). 각 후보를 별도 altLayer에 회색 마커로 추가하고 selectAlt(placeId) 클릭 핸들러 부착. renderDetail은 stop.time === null일 때 시간 라벨을 "후보" 텍스트로 대체하고 이동 카드를 생략. 데이터는 4개 병렬 subagent가 /tmp/alt-patches/에 patch JSON 저장 후 한 번에 머지.

코드 위치

  • assets/app.js:122-181 (state.altLayer, haversineKm, altCandidatesForDay, selectAlt)
  • assets/app.js:222-244 (renderDay alt 마커 그리기)
  • assets/app.js:401 (renderDetail 시간 라벨 분기)
  • assets/styles.css:44-66 (.alt-marker 스타일)
  • data/places.json (27개 신규 항목)

회고

itinerary 변경 0건으로 후보만 시각적으로 노출했다. 좌표 기반 1.5 km 근접 필터로 별도 day_hint 데이터 필드 없이 자동 매칭이 가능하다. 향후 alt를 추가할 때도 좌표만 정확하면 자동으로 해당 day에 나타난다. 4개 병렬 subagent가 27개 큐레이션을 약 6분에 완료했다.
