Stage 5

195 curated Korean blog links added across all 65 places, plus per-restaurant and per-activity inline links, replacing generic search buttons with direct post entry.

Curated Content

Why

The existing “Korean review search” was only three search-page entry links (Naver, Google, YouTube). In the field, that still required manual searching, sorting, and evaluating. Enough trustworthy Korean blog posts exist online that pre-curating them per location and embedding the links in the data means one tap to reach the actual post.

What

  • Schema extended: place-level curated_links: [{title, url, source, snippet?}]; restaurant-level link: {title, url, source}; activity items as string | {text, link?} (backward-compatible)
  • Validator updated: all URLs enforced https://, non-empty title/source required
  • Three new UI helpers: renderActivity, renderRestaurant, renderCuratedLink
  • Curated blog card grid (title + source + 1-line snippet); inline “Review →” links on activities and restaurants; existing Naver/Google/YouTube search grid demoted to a “Find more” fallback
  • 65/65 places: 3 curated links each (195 total); 40/41 main restaurants: 1 link each; 48 main-location activities wrapped as objects

How

Phase A: schema + UI + 3 Day 1 main places to verify the pattern. Phase B: 7 background subagents dispatched in parallel by day and category (Day 2 main, Day 3 main, Day 4–5 main, Day 1 alt, Day 2 alt, Day 3 alt, Day 4 alt). Each subagent ran WebSearch for Korean blogs, curated trustworthy posts, and saved a patch JSON to /tmp/curated-patches/. A single merge script applied all 7 patches to places.json with no conflicts (subagents were read-only).

Code Locations

  • scripts/validate-data.mjs:20-39 (validateCuratedLink helper)
  • scripts/validate-data.mjs:84-115 (curated_links / activities / restaurants optional validation)
  • assets/app.js:28-66 (renderActivity, renderRestaurant, renderCuratedLink)
  • assets/app.js:222-265 (renderDetail curated section + fallback search grid)
  • data/places.json (all 65 places with curated_links)

Retrospective

Seven parallel subagents finished curating 62 locations in roughly 6 minutes of wall-clock time. The patch-JSON + single-writer merge pattern eliminated conflicts. Optional schema fields meant zero regressions across the 65-entry dataset during population. The “Find more” fallback grid was preserved so users can still search when curated links feel insufficient.
