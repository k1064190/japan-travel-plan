Stage 9

36 restaurant alt-candidates added with Tabelog ratings and review counts, plus a shared rating schema and inline display in both the alt-place header and main-location restaurant cards.

Restaurant Ratings

Why

Stage 8 added alt candidates to the map, but restaurants were underrepresented and, critically, there was no trust signal to help users decide whether a place was worth visiting. Seeing “Tabelog 3.8, 2,000 reviews” inline while browsing candidates on a phone makes the go/no-go decision much faster.

What

  • Schema: optional place.rating / place.review_count / place.rating_source on any place; same three fields on place.restaurants[] entries; shared validateRating helper in the validator
  • UI: renderRating(obj) helper renders “★ 4.3 · 1,234 reviews (Tabelog)” inline — only fields that are present are shown; amber-600 color, small font
  • Rating shown below name_jp in alt-place detail headers and inside restaurant card type lines in main-location detail panels
  • Data: 36 new restaurant alt entries across Day 1–4 areas (ramen, okonomiyaki, takoyaki, sushi, yakiniku, kushikatsu, udon, soba, katsu, dessert, izakaya, curry), most with Tabelog ratings; 1 sample main-restaurant rating injected to verify both UI paths

How

Schema and sample (1 entry) first to confirm inline display, then 4 parallel subagents for Day 1–4 restaurant candidates. Each subagent saved results to /tmp/restaurant-patches/dayN-food.json; a single merge script applied them, deduplicating Day 1 and Day 3 overlaps (ichiran_dotonbori and others). One http:// URL was auto-upgraded to https://. Tabelog scores are shown with a (Tabelog) source label because the 3.0–4.0 distribution can confuse Korean users expecting a 5-point scale.

Code Locations

  • scripts/validate-data.mjs:20-37 (validateRating helper)
  • scripts/validate-data.mjs:115-128 (called on place and restaurants[])
  • assets/app.js:37-46 (renderRating helper)
  • assets/app.js:48-67 (renderRestaurant with rating inline)
  • assets/app.js:416 (renderDetail header rating)
  • data/places.json (36 new alt restaurant entries + 1 sample main-restaurant rating)

Retrospective

Optional schema fields meant zero regressions across the existing 95-entry dataset. Four parallel subagents produced 36 restaurant curations in ~7 minutes. The Tabelog source label is a deliberate UX choice: the 3.0–4.0 distribution is meaningful to Japanese users but can read as “low” to Korean users without context.
