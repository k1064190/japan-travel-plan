Stage 2

Full data populated: 65 places across 5 days, with verified coordinates, restaurant picks, and a TDD validator to guard schema integrity.

Data

Why

Stage 1 confirmed the wiring works; now real information must be filled in. The standard required: accurate coordinates, sourced references, restaurant picks for main stops, and live review-search links — enough for a traveler to follow on-site.

What

  • docs/stage-2/place-catalog.md — master list of 65 places (id, Korean name, main/alt classification)
  • data/places.json — 65 places total: Day 1: 12 (3 main + 9 alt); Day 2: 21 (5 main + 16 alt, Kyoto); Day 3: 26 (6 main + 20 alt, Osaka); Day 4: 4 (1 main + 3 alt, USJ); Day 5: 2 utility stops (departure)
  • data/itinerary.json — Day 1 through Day 5, 21 stops, every stop carries transit_from_prev and duration_minutes
  • scripts/validate-data.mjs — schema / Japan coordinate bounds / place_id cross-references / URL safety / main-to-restaurants rule (utility tag exception) validation
  • tests/validate-data.test.mjs — 11 test cases, all passing
  • package.json — npm test and npm run validate scripts

How

Each day dispatched to a separate subagent with its slice of the master catalog and a strict JSON schema prompt. Coordinates extracted via WebSearch from Wikipedia and official sites, then verified against per-city bounding boxes. Results merged into places.json by the implementer and committed. TDD order: tests written first, validator written to pass them, then real data run through validator. One policy gap surfaced immediately — utility stops (airport, hotel, check-out) must not require restaurant entries — and was codified as an explicit rule and an additional test.

Code Locations

  • data/places.json (65 entries)
  • data/itinerary.json (5 days, 21 stops)
  • scripts/validate-data.mjs — validate(places, itinerary)
  • tests/validate-data.test.mjs (11 tests)
  • package.json — test, validate scripts
  • docs/stage-2/place-catalog.md (master list)

Retrospective

  • Day-level dispatch completed 60+ place records efficiently; all coordinates passed Japan-wide and city-level bounding box checks.
  • Restaurant picks for main stops are limited to verified establishments: Okutan Kiyomizu (est. 1635), Kushikatsu Daruma original, % Arabica, Saryo Tsujiri, and others — findable by name.
  • TDD validator caught the utility-tag policy gap on first run; the fix was immediately promoted to a permanent test (test 11 of 11).
  • Nippombashi and Den Den Town share nearly identical coordinates by design — they refer to the same street at different granularities.
  • City boundary checks (Kyoto 34.5-35.5 lat, Osaka 34.5-34.8 lat) blocked coordinate errors before they reached the map.
  • Deferred to Stage 3: mobile tab activation and .numbered-marker.active pulse behavior.
